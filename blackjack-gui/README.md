# Blackjack Item Battle

Pythonで実装した，アイテム要素付きブラックジャックGUIアプリです．
元のコンソールゲームを，GitHub提出・就活ポートフォリオで説明しやすい構成に整理しました．

## 概要

プレイヤーとCPUがブラックジャック形式で対戦するゲームです．通常のHit / Standに加えて，手札交換やカード選択などのアイテム機能を実装しています．

## 主な機能

- GUIによるカード，スコア，アイテム残数，勝利数の表示
- Hit / Stand による基本操作
- Exchange: CPUとプレイヤーの手札を交換
- 1More: 2枚の候補カードから1枚を選択
- Restart: 現在のラウンドを最初からやり直し
- EXlife: 敗北時に自動で復活
- 勝利数や21達成に応じたアイテム報酬

## 使用技術

- Python 3.10以上
- Tkinter（Python標準ライブラリ）
- unittest / pytest互換のテストコード

追加ライブラリなしで実行できます．

## 実行方法

```bash
python main.py
```

macOSやLinuxでTkinterが入っていない場合は，Pythonのインストール方法によって別途Tkinterの追加が必要な場合があります．

## テスト実行

pytestを使う場合は以下です．

```bash
python -m pytest
```

pytestを入れていない環境でも，ゲーム本体は追加ライブラリなしで動作します．

## 操作方法

| ボタン | 内容 |
|---|---|
| Hit | カードを1枚引く |
| Stand | プレイヤーの行動を終了し，CPUが勝敗判定までカードを引く |
| Exchange | CPUとプレイヤーの手札を交換する |
| 1More | 2枚の候補カードから1枚を選んで追加する |
| Restart | 現在のラウンドをやり直す |
| Next Round | ラウンド終了後に次のラウンドへ進む |
| New Game | 勝利数とアイテム数をリセットして新規開始する |

## 設計上の工夫

### 1．ゲームロジックとGUIの分離

`game.py` はTkinterに依存していません．カード処理，スコア計算，勝敗判定，アイテム処理を `BlackjackGame` クラスに集約しています．
GUI側の `gui.py` は，ボタン操作に応じて `BlackjackGame` のメソッドを呼び出し，返された状態を画面に反映する役割に限定しています．

### 2．状態管理

ゲームの進行状態を以下のように分けています．

- `PLAYER_TURN`
- `CHOOSE_ONE_MORE`
- `ROUND_END`
- `GAME_OVER`

状態に応じて，押せるボタンを制御しています．これにより，不正なタイミングでの操作を防いでいます．

### 3．拡張しやすい構成

GUIをTkinterからWebアプリやPygameに変更する場合でも，`game.py` を再利用しやすい構成にしています．
今後，戦績保存，難易度設定，カード画像の導入などを追加しやすくしています．
